CS-network-모두의 네트워크-01-네트워크 첫걸음
2023-01-24 20:17:35 # CS-네트워크 # 모두의 네트워크 #네트워크

모두의 네트워크 책을 읽고 요약, 정리한 글입니다.

1. 네트워크 구조

1.1 네트워크란?

  • 네트워크라는 용어는 컴퓨터뿐만 아니라
    • 사람과 사람의 네트워크, 도로와 철도의 네트워크, 물류 네트워크와 같이 다양한 네트워크가 존재
  • IT에서는 컴퓨터 2대 이상이 연결되어 있는 컴퓨터 네트워크를 의미

1.2 패킷 = 택배

  • 네트워크나 인터넷에서 데이터를 주고받으려면 규칙이 있어야 한다.
    • e.g. 우리가 쇼핑몰을 볼 때 옷의 사진과 옷에 대한 설명
  • 이 사진과 문자를 보여주기 위해 필요한 규칙이 있는데, 이 규칙에는 패킷(packet)을 이용

패킷은 컴퓨터 간의 데이터를 주고받을 때 네트워크를 통해 전송되는 데이터의 작은 조각

  • 패킷은 큰 데이터가 있더라도 작게 나누어서 보내는 게 규칙

  • 작게 나누지 않고 큰 상태 그대로 보내면 번거롭지도 않고 오히려 더 편할 수 있지만,

  • 큰 데이터를 한번에 보내면

    • 그 데이터가 네트워크의 대역폭을 너무 많이 차지(점유) 해서 다른 패킷의 흐름을 막을 위험이 존재

  • e.g. 네트워크는 도로와 같다.

    • 큰 데이터를 네트워크에 보내면 도로와 마찬가지로 네트워크에 정체가 발생함

  • 작게 분할한 패킷은 목적지에서 원래 상태로 되돌리는 작업이 필요

    • 목적지에 도착한 패킷들은 전송한 순서대로 도착하지 않음
    • 네트워크가 지연되어 늦게 도착하거나 패킷이 누락되기도 함
    • 그래서 송신 측에서 수신 측으로 패킷을 보낼 떄는 각 패킷에 수선대로 번호를 붙여서 보냄
    • 그러면 번호에 맞춰 정렬하면 늦게 도착한 패킷도 원래 위치로 돌아갈 수 있음

2. 정보의 양을 나타내는 단위

2.1 비트와 바이트

모든 컴퓨터는 숫자 0과 1만을 다루는데, 그 0과 1의 집합을 **디지털 데이터(digital data)**라고 함

  • 비트(bit) : 0과 1의 정보를 나타내는 최소 단위
  • 바이트(byte) : 0과 1을 표현하는 1비트는 8개를 모아 표시 가능

2.2 문자코드

컴퓨터는 모든 것을 0과 1의 집합으로만 다룬다고 했다. 그러면 문자는 어떻게 입력되나?

  • 숫자와 문자의 대응표를 미리 만들어 두면 된다.
  • 이 대응표를 문자 코드(character code)라고 한다.
  • 문자 코드중 하나인 ASCII(아스키) 코드는 알파벳, 기호, 숫자 등을 다룰 수 있는 기본적인 문자코드

3. 랜과 왠

네트워크는 접속할 수 있는 범위에 따라 크게 두 가지 종류로 나눌 수 있다.

  • 랜(LAN) : 지리적으로 제한된 곳에서 컴퓨터와 프린터를 연결할 수 있는 네트워크
    • e.g. 사무실, 가정
  • 왠(WAN) : 지리적으로 넓은 범위에 구축된 네트워크
    • e.g. **인터넷 서비스 제공자(ISP)**가 제공하는 서비스를 사용하여 구축된 네트워크
      • ISP = 인터넷 사용 서비스 사업을 하고 있는 KT, U+, SK브로드밴드와 같은 통신 회사
    • e.g. 부산 사무실과 서울 사무실을 연결
    • 즉, 랜과 랜을 연결한 것

4. 가정에서 하는 랜 구성

가정에서 구성하는 네트워크는 주로 랜

  • 인터넷을 사용하기 위해서는 결정해야 할 2가지가 있는데, **인터넷 서비스 제공자(ISP)**와 인터넷 회선이다.
  • 현재는 대부분 광랜(광케이블 X)을 사용하는 사람이 많다.
  • cf. 인터넷 공유기는 인터넷 서비스 제공자와 네트워크를 연결
  • 보통 위 그림처럼 인터넷 공유기를 중심으로 내부 인터넷망(사설망)을 구성하고, 다양한 기기를 연결함
  • 연결 방식은 크게 유선 랜과 무선 랜으로 나뉨
    • 유선 랜(유선) : 랜 케이블이 필요
    • 무선 랜(무선) : 랜 케이블이 필요X, (e.g. WIFI)

5. 회사에서 하는 랜 구성

5.1 작은 기업

  • 소호 기업(SOHO)는 Small Office/Home Office의 약어이며, 소호기업은 소규모 회사를 의미

가정에서 랜 구성과 다르게 **DMZ(DeMilitarized Zone)**라는 네트워크 영역이 있음

  • DMZ : 외부에 공개하기 위한 네트워크
    • 서버를 공개하는데 주로 웹 서버, 메일 서버, DNS 서버를 공개
    • 웹 사이트를 불특정 다수의 외부 사용자에게 공개하려면 웹 서버를 외부에 공개
    • 외부 사용자와 메일을 주고받으려면 메일 서버를 외부에 공개
    • 외부에서 도메인 이름을 사용하여 회사의 서버에 접속하려면 DNS 서버를 외부에 공개

회사에서는 서버를 운영하는 방법은 3가지가 있다

  1. 서버를 사내에 설치하거나
    • 사내에서 서버를 운영하는 경우에는 회사 내에 서버 장비실을 두고 그곳에 랙(선반)을 설치한다.
    • cf. 구글에 “서버 렉” 검색
    • 사내 또는 데이터 센터에 서버를 두고 운영하는것을 **온프레미스(on-premise)**라고 한다
  2. 데이터 센터에 두거나
  3. **클라우드(cloud)**에 둔다.
